# CountElementsAPP

## Описание проекта

Проект разработан в рамках лабораторной работы №3 по дисциплине **«Алгоритмизация и программирование»**. Реализует вычисление суммы элементов массива двумя способами: итеративно и рекурсивно (метод «разделяй и властвуй»). Позволяет сравнить время выполнения и количество элементарных операций, а также наглядно увидеть результаты в графическом интерфейсе.

## Постановка задачи (Вариант 14)

Реализовать вычисление суммы элементов массива:
- **итеративно** – последовательный перебор;
- **рекурсивно** – методом «разделяй и властвуй».

Входные данные: массив целых чисел размером от **100 до 100000**. Сравнить алгоритмы по времени выполнения и глубине рекурсии.

## Функциональность

- Генерация массива случайных чисел заданного размера (диапазон значений 100–99999).
- Отображение сгенерированного массива в удобном строковом формате.
- Итеративное вычисление суммы с замером времени и подсчётом элементарных операций (шагов).
- Рекурсивное вычисление суммы с аналогичными метриками.
- Вывод результатов в таблицу (название алгоритма, время в мс, количество шагов) и отдельных меток для сумм.
- Обработка ошибочных ситуаций:
  - некорректный ввод (не число, отрицательное или нулевое значение);
  - превышение максимального размера массива (более 100000);
  - попытка вычислить сумму до генерации массива.

## Технологии

- **Язык:** C# (7.0), (UI-разработка: HTML, CSS)
- **Платформа:** .NET 10.0
- **Интерфейс:** Windows Forms
- **Тестирование:** MSTest

## Архитектура проекта

Решение состоит из трёх проектов:

1. `ClassLibrary` - Библиотека классов с основным классом `MassSum`, реализующим алгоритмы. 
2. `CountElementsAPP` - Windows Forms приложение для взаимодействия с пользователем. 
3. `TestProject` - Проект модульных тестов (MSTest) для проверки корректности работы. 

### Основные классы

- **`MassSum`** – инкапсулирует массив, флаги ошибок и методы:
  - `GenerateArray(int countElements)` – создание массива.
  - `PrintArray()` – строковое представление.
  - `IterativeCalc(out double time, ref long steps)` – итеративная сумма.
  - `RecursionCalc(out double time, ref long steps)` – рекурсивная сумма.
  - `RecursionRange(...)` – вспомогательный рекурсивный метод.
- **`CountElementsF`** – главная форма, обрабатывающая события и отображающая данные.

## Использование

1. Введите желаемый размер массива (целое число от 1 до 100000) в поле **«Количество элементов»**.
2. Нажмите кнопку **«Сгенерировать»**. Сгенерированный массив появится в текстовом поле.
3. Нажмите кнопку **«Вычислить сумму»**. В таблице отобразятся время выполнения и количество шагов для каждого алгоритма, а под таблицей – сами суммы.
4. При возникновении ошибки (неверный ввод, превышение размера и т.п.) появится окно с сообщением.


## Тестирование

Проект содержит **15 модульных тестов**, написанных с использованием **MSTest**. Тесты покрывают:
- корректность генерации массива и обработку ошибок;
- правильность суммирования на типичных и граничных данных;
- подсчёт элементарных операций (шагов);
- реакцию на ошибочные состояния (флаг `error`).


## Результаты

Экспериментальное исследование подтвердило теоретические оценки сложности:
- оба алгоритма имеют линейную временную сложность **O(n)**;
- итеративный метод выполняется в **2–3 раза быстрее** рекурсивного из-за меньших накладных расходов;
- рекурсивный алгоритм требует дополнительной памяти **O(log n)** для стека вызовов, что при максимальном размере 100000 даёт глубину около 17 (безопасно).
